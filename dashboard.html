    Create Microsoft 365 Account

    Microsoft 365 admin requires you to create accounts in the admin center directly. Below is the data formatted for M365's CSV bulk-import. Choose your path:

    M365 bulk-import format
    Format notes: UPN is firstname.lastname@go.team (lowercase). Department is Internal for GoTeam hires, otherwise Morning or Night based on PH shift start time. License SKU is O365_BUSINESS_ESSENTIALS for Internal, DESKLESSPACK for client-assigned. Passwords are 8 alphanumeric chars (no I l 1 O 0), regenerated each time the modal opens — save the CSV before closing. PowerShell script also enforces MFA by adding each user to the security group MFA-Enforced Users (configurable; must exist and be targeted by a Conditional Access policy). When PowerShell mode is active, an extra Open PowerShell as Admin button appears — click it to copy an elevation command to your clipboard. Press Win+R, paste with Ctrl+V, hit Enter, accept UAC — and an elevated PowerShell window opens.

    Create Teramind Account

    Teramind account provisioning. The CSV is in Teramind's bulk-import format; the PowerShell script calls the Teramind REST API. Edit the API base URL and key at the top of the script before running.

    Standard CSV format (21 columns) Counter: 25643 (next: GoTeam25644)
    Format notes: Standard CSV (comma-separated, quoted fields, 21 columns) matching your teramind.go.team bulk-import format. Opens cleanly in Excel. Email is firstname.lastname@go.team. ROLE = user. PASSWORD is sequential GoTeam<N> starting from where the last batch ended (use the pencil icon to set the counter). Each modal open advances the counter by N — save the CSV before closing or those numbers are lost. DEPARTMENT is the client company name. Stream/record/web policy flags default to your standard config (stream+record screen=yes, audio=no, web login=yes, GPS=yes).

    Refresh Dashboard from Email

    The dashboard's data was last captured on May 8, 2026. To pull the latest CLEAR-A-PATH emails (new starters, withdrawals, reschedules) and rewrite this file, just ask Claude in your conversation:

    refresh the dashboard

    What happens when you ask:

    1. Claude searches your Outlook for CLEAR-A-PATH emails received since the last refresh
    2. Reads each new original email and its PDF attachment to extract Position, Nickname, PH Shift
    3. Scans reply threads for withdrawals and reschedules
    4. Re-reads the May+ sheets of the Starters Tracker for fresh Employee IDs
    5. Rewrites dashboard.html with the merged data and a new "Data as of" date
    6. Reload this page in your browser to see the changes
    Note: Your local edits (status changes, MS365 marks, manually-added hires) are saved in browser storage and survive a refresh. Only the seed data — the email-derived hires — gets rewritten.
